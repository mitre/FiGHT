================================================================================
T1499.002: Base station flood with fictitious access requests
Architecture Segment: 
----------------------------------------
Procedure Examples
----------------------------------------
 Access request flooding: Adversary transmits an overwhelming number of access requests over Random Access CHannel (RACH) to degrade the ability of legitimate UE to obtain access. May be done via a compromised UE or a SDR running OAI-5G modified software.
----------------------------------------
Mitigations
----------------------------------------
FGM1502: Increase RACH resources.
----------------------------------------
Pre-Conditions
----------------------------------------
Transmit to gNB with sufficient power to succeed in flooding.: Adversary must be able to transmit to gNB with sufficient power to be received.
----------------------------------------
Critical Assets
----------------------------------------
RACH: Adversary targets access requests in Random Access CHannel (RACH)
----------------------------------------
Detection
----------------------------------------
FGDS5007: Excessive number of access requests received at gNB.
----------------------------------------
Post-Conditions
----------------------------------------
Less service for legitimate UEs: Legitimate UEs have low probability of successfully requesting access
----------------------------------------
References
----------------------------------------
European Union Agency for Cybersecurity (ENISA): “ENISA Threat Landscape for 5G Networks” Report, December 2020.: https://www.enisa.europa.eu/publications/enisa-threat-landscape-report-for-5g-networks
